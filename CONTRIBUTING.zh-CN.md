# 贡献指南

[English](CONTRIBUTING.md) | [简体中文](CONTRIBUTING.zh-CN.md)

感谢你对 Agent 学习路线图的关注！欢迎任何形式的贡献。

## 贡献方式

### 🐛 内容纠错

如果你发现错别字、链接失效、概念表述不准确等问题：

1. Fork 本仓库
2. 在相应的课程文件中修复
3. 提交 PR，标题使用 `[fix]` 前缀，例如：`[fix] 修正课程三中的 API 调用示例`

这类修复**不需要**提前开 Issue，直接提 PR 即可。

### 💡 内容补充与优化

如果你想新增章节、重写某部分内容、或大幅调整课程结构：

1. 先开 Issue 描述你的想法
2. 等待维护者确认后再开始开发
3. 避免做了大量工作后发现方向不一致

Issue 标题建议使用 `[提议]` 前缀，例如：`[提议] 课程五增加 Function Calling 实战章节`。

### 🌍 翻译

本项目采用双语目录组织：

- `courses/`：简体中文课程内容；
- `courses-en/`：英文课程内容。

如果你希望贡献英文或其他语言翻译：

1. 先开 Issue 声明你要翻译的课程
2. 尽量按照现有语言目录结构创建或更新对应文件
3. 保持原文技术术语的准确性
4. 保持链接、示例和图示与原版本一致

### 🔧 代码示例

代码示例遵循以下规范：

- **语言**：优先使用 Python（课程主线语言）
- **依赖**：尽可能使用标准库或主流框架（LangChain、OpenAI SDK、AutoGen 等）
- **可运行**：示例应标注所需依赖和运行环境
- **注释**：关键步骤应有中文注释说明

## PR 规范

- PR 标题简明描述改动内容
- PR 描述中说明改动原因和影响范围
- 一个 PR 只做一件事（修复一个错别字 vs 新增一个章节 → 分开提）

## 沟通渠道

- **问题与讨论**：GitHub Issues
- **学习交流**：GitHub Discussions

## 行为准则

- 保持尊重和包容
- 聚焦于内容本身，避免人身攻击
- 欢迎不同观点，但需要有建设性
