# 参考思考

本目录收录与课程主线互补的深度讨论文档。它们不是课程章节，不教具体的实现方法，而是围绕课程背后的本质问题展开——模型到底是怎么工作的、Agent 的任务完成如何保障、产品设计中的取舍。

这些内容适合在学完课程主体后阅读，帮助你建立对 Agent 更立体的理解：既不高估模型的能力，也不低估工程机制的价值。

## 文档

- **[当模型停下来时——大模型工作机制深度讨论](llm-mechanism.md)**  
  从四个递进问题理解大模型：它怎么「知道」任务完成了？怎么拆解任务？怎么处理没见过的问题？以及，它有意识吗？  
  *对应课程：课程二（范式演进）、课程三（最小闭环）、课程五（场景增强）*

- **[用户说「你先试试」——模糊需求下任务完成的工程思考](task-completion-engineering.md)**  
  模型没有真正的「完成感」，用户也说不清「怎么样算完成」。在这种双重模糊下，系统如何保障任务质量？从信号驱动验证、计划确认交互、到分层保障的工程实践。  
  *对应课程：课程五（Reflection）、课程六（Harness）、课程七（产品化）*
